P2 told me which RAG configuration works best: semantic chunking, OpenAI embeddings, Cohere reranking. P5 is about turning that into something I can actually deploy.

The plan is a FastAPI service with hybrid search (dense vectors + BM25), Cohere cross-encoder reranking, and ChromaDB for persistence. Containerized with Docker. The interesting part is not the retrieval logic. It is the operational layer: latency budgets, memory footprint, graceful degradation when the vector store is unavailable, and monitoring hooks for observability.

Most RAG tutorials end at "retrieval works in a notebook." This project starts where they stop.
