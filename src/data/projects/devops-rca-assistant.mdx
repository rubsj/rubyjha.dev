Production incidents are diagnosed through a mental model built from logs, metrics, alerts, recent deployments, and institutional memory. This project codifies that model as a multi-agent CrewAI system: a log analysis agent identifies anomaly patterns and error spikes; a metrics agent correlates CPU, memory, and latency curves against the incident timeline; an alert agent maps triggered alerts to known failure signatures; and an RCA synthesis agent produces a structured diagnosis with confidence levels and recommended remediation steps.

The system is designed for the first 15 minutes of an incident — the highest-stress, highest-stakes window where having a structured hypothesis quickly reduces mean time to resolution. The output is a structured incident report with ranked hypotheses, supporting evidence from each data source, and suggested next diagnostic steps for the on-call engineer to validate.
